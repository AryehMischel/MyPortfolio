360 image classifier / viewer GithuB DeMO
• WebXR 360 image viewer.

• Drag and drop 360 images and the application will do the rest.

• Automatically supports all common 360 image formats.

• Auto detects and processes image based on detected format.

• VR support
‍
• Supports Stereoscopic Equirectangular and Stereoscopic Cubemaps.

• Experimental integration with the Google Drive API

WeBXR Layers GithuB VR DeMO
• View high-resolution 360 images with incredible clarity in VR.

•WebXR Layers is an experimental API that extends Meta's timewarp functionality.
‍
This API is critical for WebXR
because timewarp layers;

• Boost Performance,
•Create high visual fidelity,
• Increase Battery Life,
• Are compatible with Compressed GPU textures.

Compressed GPU texture Pipeline
• A Linux docker application that creates Compressed GPU textures from standard image formats. It uses the AWS SDK to receive process and return user images to S3. Internally, it uses KTX Tools to create and compress the textures.
‍


Putting it all together

Here I am describing the state of the full project.
The front end is written in JavaScript and WebGL (In Three.JS) .
The backend is a NodeJS and Express with passport.js for OAuth (Google sign-in strategy).
Currently using Express sessions with MySQL, but will likely shift to JWT's
I am using a MySQL database hosted by AWS RDBS.
The final hosting, deployment, scaling, networking, and data storage will all be done within the AWS ecosystem (Elastic Beanstalk, S3, CloudFront)
The current Compressed GPU texture Pipeline is also done within the AWS ecosystem (S3, SQS, ECS Fargate, AWS ECR, Docker, Bash, AWS SDK, KTX Tools)
